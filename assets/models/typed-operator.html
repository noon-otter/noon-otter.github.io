An Illustrated Framework for Model Risk Management

A Model Is a Typed Operator.

A model takes an input, runs in an environment, and returns an output. Pin the version and the rest of governance falls out as algebra: composition, parallel execution, versioning, dynamic routing — and a chain rule that lets risk propagate through any pipeline you can draw, from a Black–Scholes pricer to a tool-using agent.

The atom. Forget for a moment what a model is made of and consider what it does. It takes an input x, runs in some environment Ω, and returns an output y. Pin the version v and that triple is a contract: the same (M^v, x, Ω) always produces the same y. That is reproducibility, written down. The Black–Scholes pricer and a 70-billion-parameter language model live in the same definition.

The algebra. Five axioms — immutability, typing, closure, identity, associativity — are the precise rules under which two operators wire together without ambiguity. From them, four operations: composition chains them in series; parallel product runs them on the same input; versioning moves them through time; dynamic routing lets one model decide which other model runs. Closure guarantees the result is again a model, so the rules apply at every level.

The chain rule. Because models compose, sensitivities propagate. A change in a market feed deep inside M₁ moves the order ticket out of M₃ by the product of the local Jacobians. Where the components are smooth, this is calculus. Where they are discrete or stochastic, Lipschitz bounds replace differentiation — the structure is the same. Risk does not stop at a module boundary. It travels.

The point. A static pipeline and an agentic LLM look different until you draw them as typed operators. Then they are the same picture with different shapes in the boxes. The Black–Scholes stack has scalar inputs and a closed form; the agent has natural-language prompts and a router that makes the routing decision itself measurable. Either way: governance is bookkeeping on the triple (M^v, x, Ω). Everything else is commentary.
