A. SEQUENCE
Composition
M₂ ∘ M₁ · the chain rule
$ (M_2 \circ M_1)(x, \Omega) \;=\; M_2\!\bigl(\, M_1(x, \Omega_1),\; \Omega_2 \,\bigr) $
end-to-end sensitivity∂y/∂x = (∂M₂/∂z)(∂M₁/∂x)
Sensitivities multiply. Risk in any component flows downstream — bounded by the product of Lipschitz constants.
B. ENSEMBLE
Parallel Product
M₁ × M₂ · champion vs. challenger
$ (M_1 \times M_2)(x, \Omega) \;=\; \bigl(\, M_1(x, \Omega_1),\; M_2(x, \Omega_2) \,\bigr) $
branch sensitivities∂y/∂x = (∂M₁/∂x, ∂M₂/∂x)
Subadditive risk: ℛ(M₁ × M₂) ≤ ℛ(M₁) + ℛ(M₂). Shared Ω couples the branches.
C. EVOLUTION
Versioning
T · the higher-risk operator
$ T \,:\, (M^{v}, \Delta) \;\longrightarrow\; M^{v+1} $
version sensitivity∂y/∂v = d( y(Mv+1), y(Mv) )
A version transition is itself a model. Its magnitude can trigger automatic revalidation.
D. AGENCY
Dynamic Routing
R selects Mi · routing is itself a model
$ M(x, \Omega) \;=\; M_{R(x,\,\Omega_R)}\!\bigl(\, x,\; \Omega_i \,\bigr) $
routed sensitivity∂y/∂x = ∂Mi/∂x + ∂y/∂R
Adds a new sensitivity: ∂y/∂R. An agent that almost called a different tool carries latent risk in R.
